* 유효성 검사 (스크립트로 많이 사용)
  - 대표적으로 회원가입
      아이디에 한글쓰면? 안 됨
      비밀번호 안 썼는데 회원가입? 안 됨
      전화번호에 공일공? 안 됨

      칸에 들어가는 값이 규칙에 맞는가  검사하는 것이 유효성 검사

  유효성 검사가 왜 필요할까?
    예약을 받는다 하면 날짜, 장소, 인원 등이 정확한 값을 입력받아야 한다.

    입력받은 값들은 규칙이 있을 수 있다. 그 규칙에 부합하는지 검사하는 것이 유효성 검사.
    입력받은 값들이 적절한가는 버튼(ex. 회원가입)을 눌렀을 때


  function form_check() {
    // 언제 누가 사용할 건지 파악하기위해 이름이 필요함.
      검사 할 내용
  };

  btn.onclick = function () { }
    // 얘는 누가 언제 뭘하는지 명확하니까 이름 필요 없음


커서는 기본적으로 링크<a>와 폼에만 생긴다.
태그 안에 tabindex="0" 쓰면 커서가 이동하긴 한다.
  (ex. <legend tabindex="0">)
  근데 가급적이면 안쓴다.

  커서가 존재한다 focus
  커서가 사라지다 blur

    focus       커서가 생기다 (이벤트, 커서가 생기는 상황)
    focus();    커서야, 생겨라 [명령]
    onfocus=    커서가 생겼을 때

전화번호
  label은 input에 어떤 내용을 써야하는지 알려주는 역할
  두 번째 입력칸에 - 에 label을 붙일 순 없음 (리더기에서 빼기라고 알려줌)

  암묵적으로 약속한 것은 label은 첫 번째 입력칸에 부여하기로 함

  입력칸 하나만 두고 입력하도록 바뀜
    -코드 절감효과, 사용자도 편해짐, 접근성도 높아짐

    + 생년월일, 이메일도 같은 맥락으로 바뀜


  정규식(정해진 규칙)을 이용하여 전화번호 숫자만 입력받도록 함
    -정규식 검색 시 주의 사항 : 반드시 앞에 언어의 이름 써야함
                              ex) 자바스크립트 정규식

                              왠만한 프로그래밍언어에는 정규식 존재 + 사람이름 나옴

    정규식은 /규칙/플래그 써서 규칙을 만듦
